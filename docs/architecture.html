terraform-azure-postgresql-flexible

Executive Summary

Terraform module for deploying Azure Database for PostgreSQL Flexible Server with zone-redundant high availability, geo-redundant backups, pgBouncer connection pooling, VNet integration via delegated subnets, customer-managed key (CMK) encryption, Azure AD authentication, firewall rules, and diagnostic settings for threat detection. Supports PostgreSQL versions up to 16 with configurable maintenance windows.

1. Application connects through delegated subnet (VNet integration) or firewall-allowed IP. 2. pgBouncer pools connections for efficiency. 3. Primary server processes queries; HA standby in another zone for automatic failover. 4. Backups replicated geo-redundantly for DR. 5. Diagnostic logs shipped for threat detection analysis.

Encryption

Network Isolation

Industry Adaptation

SaaS Applications

pgBouncer for high-connection workloads, auto-grow storage, zone-redundant HA, multiple databases per server

Financial Services

CMK encryption, geo-redundant backups, Azure AD auth only, VNet-integrated with no public access

Healthcare

HIPAA-compliant configuration, 35-day backup retention, threat detection enabled, AD authentication

Analytics Platforms

Memory-optimized SKU (MO_Standard), query store for performance tuning, large storage with auto-grow

Production Readiness Checklist

  • Zone-redundant high availability enabled
  • Geo-redundant backups with 35-day retention
  • Storage auto-grow enabled
  • Threat detection diagnostics active
  • VNet integration via delegated subnet
  • Enable pgBouncer for connection pooling
  • Configure Azure AD authentication
  • Set up CMK encryption with Key Vault
  • Define maintenance window for off-peak hours
  • Tune server configurations for workload
